Visual Studio Code 偏好设置

2020-02-15 17:58
#vscode #tools

Visual Studio Code 提供了极大的弹性让我们自订想要的编辑器样式，端看使用者设定（User Setting）中，就超过 800 项设定可以让开发者自由调整，而且还有扩充套件能够增强 VS Code 的开发能力。

这篇主要是我个人偏好的设定，完整 setting.json 请参考 gist。

首先可以使用快速键 Ctrl +, 开启 settings.json 使用者设定档，或从选单列中点选档案 > 喜好设定 > 设定。

Global 常用设定

editor

1

editor.fontFamily // 设定字型偏好
  • 唯一选择 Source Code Pro
  • 连字符号字型可选择 FiraCode 或 Hasklig 择一使用(个人偏好 FiraCode)
  • 设定值："editor.fontFamily": "'Fira Code', 'Source Code Pro', Consolas, 'Microsoft JhengHei', 'Courier New', monospace",

"editor.fontLigatures": true // 开启连字符号
// FiraCode 字型有提供连字符号的功能，例如 => 会变成箭头符号
"editor.formatOnSave": false /* 存档时不进行自动排版
- 可用 Alt + Shift + F 手动执行自动排版*/
"editor.minimap.enabled": true // 在捲轴上开启 MiniMap 功能
"editor.minimap.renderCharacters": false // MiniMap 不渲染实际字元
"editor.renderWhitespace": "boundary" // 显示空白字元
"editor.renderIndentGuides": true // 显示缩排线
"editor.wordWrap": "on" // 断行显示

"files.autoSave": "onWindowChange" // 离开视窗焦点时自动储存
"files.autoGuessEncoding": true /* 猜测档案编码
- 解决 VS Code 不支援判读档案是 ASCII 编码的问题*/
"files.defaultLanguage": "markdown" // 设定预设文件语言类型,想要把 VS Code 当作预设的 Markdown 编辑器，一定不能错过这个设定
"files.insertFinalNewline": true // 新增一行作为档案结束 开发 Python 时建议开启(PEP 8)

others

1

"explorer.openEditors.visible": 0 // 设定已开启的编辑器预设是否显示

已经有「页籤」可以看已开启的档案，用 Ctrl+E 也可以快速找档案，这个功能想的多馀，建议设成 0 关闭他 terminal.integrated.shell.windows 指定使用哪种终端机

  • CMD “C:\Windows\sysnative\cmd.exe”
  • PowerShell “C:\Windows\sysnative\WindowsPowerShell\v1.0\powershell.exe”
  • Bash on Ubuntu “C:\Windows\sysnative\bash.exe”

Workspace 常用设定

我们也可以针对工作目录下做额外的设定

1

`files.exclude` // 排除指定档案
  • .spec 是前端专案常见的测试档，可藉此设定暂时隐藏，让工作目录乾淨一些
  • "files.exclude": { "**/*.spec.*": true },

Code Snippet

自订程式码片段（snippet）能够帮助我们在写程式时，快速产生程式码，加速写程式的速度。

  • Windows 版，自订程式码片段的功能位置：档案 > 喜好设定 > 使用者程式码片段
    • 自订程式码片段会放在 C:\Users\12258\AppData\Roaming\Code\User\snippets
  • Mac 版，自订程式码片段的功能位置：Code > 喜好设定 > 使用者程式码片段 程式码基本架构如下：
